modules

Chicken v5 should be a lot easier, apparently.
  • extension - packaged modules, with possibly more than one module in an extension. aka "egg"
  • import - in v4 it extends the namespace, but does not do any code loading. That is what "use" is for
  • include - works like C #include as it textually inserted the mentioned file
  • require-extension - aka "use"
  • unit - traditional compiler units, better forget this term now (for v5, I think)
  • use - aka "require-extension" (qv)
